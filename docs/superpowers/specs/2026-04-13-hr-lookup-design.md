# 인사정보 조회 기능 설계

## Context

현재 엘리시아 근태 관리 시스템은 출퇴근 기록, 달력, 엑셀 업로드 기능을 제공하지만, 직원 개인의 종합적인 인사 데이터를 조회하는 기능이 없다. 인사 데이터는 Google Sheets("Master Sheet 2.0")에 별도로 관리되고 있으며, 이를 시스템에서 직접 검색할 수 있도록 하여 관리자의 업무 효율을 높이고자 한다.

## 요구사항

- 이름을 입력하면 해당 직원의 모든 인사 데이터를 한 눈에 표시
- Google Sheets API를 통해 실시간 연동
- 기존 대시보드에 새 탭으로 추가
- ADMIN 사용자만 접근 가능
- 민감정보(주민등록번호, 휴대번호) 마스킹 처리

## 기술 스택

- Google Sheets API (`googleapis` 패키지)
- Google Cloud 서비스 계정 인증
- 5분 TTL 인메모리 캐시

- **시트 이름**: `Master Sheet 2.0`

  - 고용형태, 성별, 주민등록번호, 생년월일, 나이
  - 이메일, 내선번호, 휴대번호, 결혼유무

## 아키텍처

```
[대시보드 UI] → [GET /api/hr-lookup?name=김] → [google-sheets.ts]
   (탭)              (API Route)                  ↓
                                            [인메모리 캐시]
                                                  ↓ (cache miss)
                                            [Google Sheets API]
                                                  ↓
                                            [Master Sheet 2.0]
```

## 파일 구조

### 새로 생성하는 파일

#### 1. `src/lib/google-sheets.ts` — Google Sheets 연동 핵심 모듈

**역할**:
- Google 서비스 계정 JWT 인증
- 시트 데이터 전체 fetch 및 파싱 (행 0 = 헤더, 행 1+ = 데이터)
- 5분 TTL 인메모리 캐시
- `searchEmployeeByName(query)` — 성명 컬럼에서 부분 일치 검색
- 민감정보 서버 측 마스킹:
  - 주민등록번호: `880101-*******`
  - 휴대번호: `010-****-5678`
- 빈 컬럼 필터링 (모든 행에서 비어있는 컬럼 제외)

#### 2. `src/app/api/hr-lookup/route.ts` — API 라우트

**역할**:
- ADMIN 권한 확인 (`getAuthUser`)
- 쿼리 파라미터 `name` 필수 검증
- `searchEmployeeByName` 호출 후 결과 반환
- 에러 핸들링: 인증 미설정(503), 시트 접근 실패(502)

### 수정하는 파일

#### 3. `src/app/page.tsx` — 대시보드 메인 페이지

**변경 사항**:
- 탭 타입에 `'hrlookup'` 추가
- 탭 버튼 배열에 `'hrlookup'` 추가, 라벨: "인사정보 조회"
- 새 상태 변수: `hrSearchQuery`, `hrResults`, `hrLoading`, `hrSelectedEmployee`
- 검색 fetch 함수 추가 (Enter 키 또는 버튼 클릭으로 트리거)
- 탭 콘텐츠 렌더링 추가

#### 4. `.env` — 환경 변수 추가


## 보안

- ADMIN 전용 (API 레벨에서 role 검증)
- 주민등록번호: 앞 6자리만 표시, 뒷자리는 `*******`
- 휴대번호: 가운데 4자리 마스킹 `****`
- 서비스 계정 키는 `.env`에만 저장, `.gitignore` 확인
- 마스킹은 서버 측에서 수행 (클라이언트에 원본 데이터 전달하지 않음)

## 에러 핸들링

| 상황 | HTTP 상태 | 메시지 |
|------|----------|--------|
| 미인증 | 401 | "로그인이 필요합니다" |
| 권한 없음 | 403 | "관리자만 접근할 수 있습니다" |
| 검색어 없음 | 400 | "이름을 입력해주세요" |
| Google 설정 미완료 | 503 | "Google Sheets 연동이 설정되지 않았습니다" |
| 시트 접근 실패 | 502 | "스프레드시트에 접근할 수 없습니다" |
| 검색 결과 없음 | 200 | UI에서 "검색 결과가 없습니다" 표시 |

## 사전 준비 (사용자 작업)

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 프로젝트 생성 (또는 기존 프로젝트 사용)
3. Google Sheets API 활성화
4. IAM & Admin > 서비스 계정 > 새 서비스 계정 생성
5. JSON 키 다운로드 → `client_email`, `private_key` 추출
6. `.env`에 환경 변수 추가
7. Google Sheet에서 서비스 계정 이메일을 뷰어로 공유

## 검증 방법

1. `.env`에 Google 서비스 계정 정보 설정
2. `npm run dev`로 개발 서버 실행
3. 로그인 (ADMIN 계정)
4. 대시보드에서 "인사정보 조회" 탭 클릭
5. 이름 입력 후 검색 → 결과 카드 표시 확인
6. 동명이인 테스트: 흔한 성을 검색하여 목록 표시 확인
7. 민감정보 마스킹 확인 (주민등록번호, 휴대번호)
8. 비 ADMIN 사용자로 접근 시 차단 확인
